# 003｜Klippa Proof of Delivery OCR 页面调研

## 1. 结论速览
- Klippa 将 Proof of Delivery（POD）OCR 作为 DocHorizon 的能力之一，定位为可自动识别与抽取 POD 关键信息的 OCR/数据抽取服务，面向物流与 ERP/数据管理集成场景。[来源](https://www.klippa.com/en/ocr/logistics-documents/proofs-of-delivery/)
- 该能力支持通过 IDP 平台、API 与 SDK 进行集成，且页面明确展示低代码平台、API 集成与移动扫描 SDK 的方案入口。[来源](https://www.klippa.com/en/ocr/logistics-documents/proofs-of-delivery/)
- 关键输出包括结构化数据与 JSON（以及声明可转换为 CSV/XLSX/XML/JSON）。[来源](https://www.klippa.com/en/ocr/logistics-documents/proofs-of-delivery/)
- 明确列出可抽取字段示例（地址、订单号、签名、数量、重量等），并强调字段可定制。[来源](https://www.klippa.com/en/ocr/logistics-documents/proofs-of-delivery/)
- 部署与合规方面：提供 EU 区域 ISO 认证服务器、可选地区部署、DPA（数据处理协议），且声明处理后不保留客户数据。[来源](https://www.klippa.com/en/ocr/logistics-documents/proofs-of-delivery/)
- 作为子页补充，OCR API 页面强调 REST API 集成、24 小时内可集成、支持 150+ 语言与 20+ 输出格式（含 JSON/CSV/XML/XLS/XLSX 等）。[来源](https://www.klippa.com/en/ocr/ocr-api/)

## 2. 关键发现
### 产品/服务定位
- “Proof of Delivery OCR”被描述为自动识别并抓取 POD 关键信息的先进技术，DocHorizon 内置 AI OCR 能实现精确抽取与格式转换，便于与 ERP/数据管理系统集成。[来源](https://www.klippa.com/en/ocr/logistics-documents/proofs-of-delivery/)
- 其机器学习技术可通过 IDP 平台、API 与 SDK 自动化多种 POD 相关业务流程。[来源](https://www.klippa.com/en/ocr/logistics-documents/proofs-of-delivery/)

### 适用文档类型/场景
- 页面定义 POD 作为“证明收货完成的文件”，类似收据但用于证明交付完成。[来源](https://www.klippa.com/en/ocr/logistics-documents/proofs-of-delivery/)
